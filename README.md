Проект представляет из себя быстрый способ воссоздать архитектуру состоящую из 2 серверов (локальный + удаленный) с определенными сервисами, которые решают специфические задачи. Проект сделан прежде всего для меня, а также для людей которые хотят свой готовый self-hosted сервер из коробки с полной системой обслуживания (см. ниже).

Также стоит уточнить что разместить всё на одном мощном VPS было бы проще, однако это потребовало бы слишком больших расходов на оплату такого сервера. Поэтому я выбрал гибридный подход: ресурсоёмкие сервисы работают на локальной машине, а удалённый VPS помогает организовать безопасный доступ к ним из интернета. Либо опять же, если хватает денег - можно спокойно арендовать сервера, вместо того чтобы держать их у себя дома.

# Краткий экскурс
## Основные сервисы
- **Nextcloud:** Облачное хранилище для файлов, контактов и фото.
- **Vaultwarden:** Легковесный менеджер паролей (совместим с Bitwarden API) для хранения шифрованных данных.
- **Gitea:** Легковесная система управления версиями на базе Git
- **Navidrome:** Стриминг личной музыкальной коллекции.
- **Audiobookshelf:** Сервер для аудиокниг и подкастов с синхронизацией прогресса.

## Сеть и безопасность
- **Webnames:** Регистратор доменных имён. Используется для владения доменом и управления DNS-записями (A-записи, Wildcard), необходимыми для работы SSL. (получение сертификатов происходит именно через Webnames, так как мой домен был куплен там, подробнее ниже)
- **WG-easy + AmneziaWG:** Зашифрованный туннель между устройствами и VPS, скрывающий инфраструктуру. Доступ к сервисам возможен строго из подсети VPN `10.8.0.0/24`; также имеется веб-панель для управления.
- **AdGuard Home:** Локальный DNS-сервер с функцией блокировки рекламы и трекеров.
- **Nginx:** Выполняет роль обратного прокси, блокируя любые внешние запросы, не принадлежащие внутренней сети туннеля.
- **SSH:** Настройка доступа по ключам `ed25519` с отключением входа по паролю.
- **Certbot:** Автоматизация получения SSL-сертификатов (Wildcard) через проверку DNS-01, интегрированную с API Webnames.
- **Fail2ban:** Защита от брутфорс-атак на системном уровне.

## Архитектура
- **Docker + Docker Compose:** Развертка всех приложений в контейнерах с помощью compose файлов.

## Мониторинг и контроль
- **Prometheus & Exporters:** Сбор метрик хоста (Node Exporter), контейнеров (cAdvisor) и самого веб-сервера (Nginx Exporter).
- **Grafana:** Визуализация состояния системы и производительности сервисов.
- **Gatus:** Мониторинг аптайма локального сервера через ежеминутное опрашивание `healthcheck` эндпоинта.
- **Portainer:** Веб-интерфейс для администрирования Docker-контейнеров.
- **Loki + Grafana Alloy:** Сбор логов со всех контейнеров в одно место.

## Алертинг и коммуникация
- **Alertmanager:** Алертинг критических событий на локальном сервере.
- **Gatus:** Мониторинг и алертинг работы локального и удаленного серверов, а также проверка SSL сертификатов.
- **TG Bot Integration:** Использование TG бота для получения уведомлений от Alertmanager и Gatus.

## Резервное копирование
- **BorgBackup + Borgmatic:** Зашифрованное автоматизированное резервное копирование на диск с сжатием lz4.
- **BorgUI:** Интерфейс для удобного просмотра бэкапов.

## Автоматизация
- **Ansible:** Система управления конфигурациями для автоматического развёртывания ОС, настройки безопасности и оркестрации Docker-контейнеров.
- **Установщик на Go и Wails:** удобный установщик, который сам развернет проект.
- **Vagrant:** Быстрое развертывание виртуальных машин для тестирования Ansible плейбуков.
- **Bash скрипты:** Автоматизация определенных шагов с помощью .sh скриптов.

## Удобство пользования
- **Homepage:** Красивая домашняя страница для удобства пользования сервисами в браузере.

# Важная информация перед развертыванием проекта
1. **Поддерживаемые ОС:** Проект рассчитан на работу на Ubuntu, Debian, Arch Linux (7.1.3-arch1-2).
2. **Настройка DNS:** В DNS-панели регистратора (например, на Webnames) необходимо направить A-запись основного домена и wildcard-запись (`*`) на внутренний IP-адрес VPN — **`10.8.0.1`**. Это критически важно: весь трафик к вашим сервисам должен идти внутри защищённого туннеля AmneziaWG. Если направить домен на публичный IP-адрес сервера VPS, Nginx заблокирует внешние запросы из-за правила `deny all`.
3. **VPN для Vagrant:** HashiCorp в данный момент не работает в России, поэтому для запуска тестирования понадобится VPN. (После установки машин VPN следует выключить, либо можно включить split-tunneling на игнорирование диапазона `192.168.0.0/24`, так как иначе к тестовым виртуалкам не получится подключиться по SSH.)
4. **Сохранность бэкапов:** После настройки локального сервера стоит сохранить ключ Borg (если настраивалась система бэкапов): `borg key export /mnt/backup_storage/server_backup.borg /home/USER/borg_key_backup.txt`. (Лучше перенести этот файл в другое место, не стоит хранить его на том же сервере.)
5. **Настройка wg-easy:** Если нет подключения к VPN, то можно попробовать зайти в интерфейс wg-easy и настроить параметры. (Подробнее можно прочитать в [AmneziaWG documentation](https://docs.amnezia.org/documentation/amnezia-wg).)
6. **Ошибки при выполнении Ansible плейбука:** Во время выполнения в самом начале могут появиться ошибки при подключении под root. Если скрипт при этом не завершился и развёртывание происходит на серверах, где уже есть пользователи, указанные в `secrets.yml`, — это нормально. В таком случае скрипт должен подключиться под созданными пользователями.
7. **Гарантия работы:** Работа проекта гарантируется только на полностью чистых серверах, либо если сервер уже был настроен с помощью этого проекта. В ином случае гарантии работы нет. Также не гарантируется работа при проблемах с подключением.

# Тестирование с Vagrant
1. Установите:
    - **[Vagrant](https://developer.hashicorp.com/vagrant/install)**
    - **[VirtualBox](https://www.virtualbox.org/wiki/Downloads)**
2. На основной системе в корне проекта выполните `vagrant up` для создания серверов для тестирования.
    - На вопрос **Which interface should the network bridge to?** нужно выбрать тот интерфейс, через который хост выходит в интернет.
3. При развертывании на сервера в secrets.yml пароль и имя пользователя должно быть `vagrant`
4. Чтобы остановить виртуальные машины, можно использовать `vagrant halt`.
5. Чтобы полностью удалить машины, можно использовать `vagrant destroy -f`.

3. Выполните команду `wails build` (может отличаться, лучше следовать документации Wails)
