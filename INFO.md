# Доп. информация по проекту
1.  **Поддерживаемые ОС:** Проект рассчитан на работу на Ubuntu, Debian, Arch Linux (7.1.3-arch1-2).
2.  **Настройка DNS:** В DNS-панели регистратора (например, на Webnames) необходимо направить A-запись основного домена и wildcard-запись (`*`) на внутренний IP-адрес VPN — **`10.8.0.1`**. Это критически важно: весь трафик к вашим сервисам должен идти внутри защищённого туннеля AmneziaWG. Если направить домен на публичный IP-адрес сервера VPS, Nginx заблокирует внешние запросы из-за правила `deny all`.
3.  **VPN для Vagrant:** HashiCorp в данный момент не работает в России, поэтому для запуска тестирования понадобится VPN. (После установки машин VPN следует выключить, либо можно включить split-tunneling на игнорирование диапазона `192.168.0.0/24`, так как иначе к тестовым виртуалкам не получится подключиться по SSH.)
4.  **Сохранность бэкапов:** После настройки локального сервера стоит сохранить ключ Borg (если настраивалась система бэкапов): `borg key export /mnt/backup_storage/server_backup.borg /home/USER/borg_key_backup.txt`. (Лучше перенести этот файл в другое место, не стоит хранить его на том же сервере.)
5.  **Настройка wg-easy:** Если нет подключения к VPN, то можно попробовать зайти в интерфейс wg-easy и настроить параметры. (Подробнее можно прочитать в [AmneziaWG documentation](https://docs.amnezia.org/documentation/amnezia-wg).)
6.  **Ошибки при выполнении Ansible плейбука:** Во время выполнения в самом начале могут появиться ошибки при подключении под root. Если скрипт при этом не завершился и развёртывание происходит на серверах, где уже есть пользователи, указанные в `secrets.yml`, — это нормально. В таком случае скрипт должен подключиться под созданными пользователями.
7.  **Гарантия работы:** Работа проекта гарантируется только на полностью чистых серверах, либо если сервер уже был настроен с помощью этого проекта. В ином случае гарантии работы нет. Также не гарантируется работа при проблемах с подключением.

### Настройка удобного просмотра бэкапов
1.  Убедитесь, что диск для бэкапов подключён к серверу.
2.  Укажите UUID диска в `secrets.yml`. (Можно посмотреть с помощью `sudo blkid`.)
3.  После разворачивания локальной архитектуры зайдите в Borg UI (имя пользователя всегда `admin`) и выполните следующие шаги:
    1.  Выберите импорт репозитория (укажите имя `/mnt/backup_storage/server_backup.borg`).
    2.  Выберите **Observability Only** и **Read-only storage access** (бэкапами занимается borgmatic, Borg UI — только для просмотра).
    3.  Укажите passphrase, которая была задана в `secrets.yml`.
    4.  Выберите тип сжатия `lz4`.
    5.  Импортируйте репозиторий.
4.  **Напоминалка:** Бэкапы через Borg UI настраивать ни в коем случае нельзя!
5.  Для включения метрик нужно перейти в **Settings** -> **System** -> **Metrics Access** и выключить параметр "Enable /metrics endpoint".

### Настройка Portainer
-   **Добавление контейнеров удалённого сервера к Portainer:**
    1.  Выберите пункт **Environments**.
    2.  Нажмите **Add environment** (Docker Standalone).
    3.  Выберите вариант подключения: **Agent**.
    4.  В поле **Environment URL** укажите: `10.8.0.1:9001`.

### Настройки автономности локального сервера
1.  Для того чтобы сервер включался обратно после отключения электричества, в BIOS нужно найти следующие настройки (пишу, как они назывались у меня, имена могут отличаться):
    1.  **State After G3:** нужно выбрать `S0 State`.
    2.  **Wake system from S5:** `Disabled`.

### Тестирование с Vagrant
1.  Установите:
    -   **Vagrant**
    -   **VirtualBox**
2.  На основной системе в корне проекта выполните `vagrant up` для создания серверов для тестирования.
    -   На вопрос **Which interface should the network bridge to?** нужно выбрать тот интерфейс, через который хост выходит в интернет.
3.  В остальном всё по стандартному гайду.
4.  Чтобы остановить виртуальные машины, можно использовать `vagrant halt`. (Остальные команды можно посмотреть с помощью `vagrant --help`.)

## Пользование сервисами
1.  **Вход на портал:** После успешного запуска перейдите по вашему домену на главную страницу (веб-портал), предварительно включив VPN-клиент AmneziaWG на вашем устройстве.
2.  **Первичная настройка приложений:** Для сервисов Navidrome, Audiobookshelf, Vaultwarden, Portainer, Gitea при первом открытии необходимо пройти быструю процедуру создания аккаунта администратора. Для доступа к системной панели администрирования Vaultwarden используется токен, заданный в переменной `secret_vaultwarden_password`.
3.  **Автоматическое развертывание:** Сервисы Nextcloud, Grafana и другие настраиваются автоматически. Данные для первого входа в них соответствуют глобальным переменным `admin_user` и `admin_password` из вашего `secrets.yml`.
4.  **Рекомендованные приложения для повседневного использования:**
    -   Облачное хранилище — *Nextcloud*
    -   Менеджер паролей — *Bitwarden*
    -   Управление DNS-сервером — *AdGuard Home Manager*
    -   Музыкальная коллекция — *Tempo*
    -   Аудиокниги и подкасты — *Audiobookshelf*
    -   VPN-клиент — *AmneziaWG / WG Tunnel*
5.  **Прямое подключение по SSH внутри защищённого VPN-туннеля:**
    -   К удалённому серверу (VPS): `ssh user@10.8.0.1`
    -   К локальному серверу: `ssh user@10.8.0.2`
    -   *(Обратите внимание: эти приватные IP-адреса будут доступны для подключения только тогда, когда на вашем текущем рабочем устройстве — ноутбуке или смартфоне — запущен и активен VPN-клиент AmneziaWG.)*
6.  **Использование папок** (при переносе проекта главное — скопировать эти папки):
    -   В папке `~/ServeHub-2/apps-data` (есть как на локальном, так и на удалённом серверах) хранятся данные приложений. При переносе проекта её важно сохранить.
    -   В папке `~/PersonalData` хранятся все пользовательские данные: документы, фото, видео, музыка и т. д.
