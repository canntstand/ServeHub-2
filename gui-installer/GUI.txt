1. пользователь качает .exe, запускает
2. проверка есть ли docker и docker compose, если нет - ошибка (+ кнопка проверить заново), если есть - то кнопка продолжить
3. Копирование репозитория рядом со скриптом через git
4. Форма заполнения secrets.yml, (просто переменная - рядом описание - и место ввода)
5. После нажатия продолжить в пред. шаге все что ввел пользователь должно показываться в окошке с просьбой скопировать и проверить (кнопка продолжить + еще раз спросить точно ли все готово + кнопка переделать) 
6. Полученные данные должны парситься в ./репозиторий/ansible/vars/secrets.yml превращаясь в yaml файл
7. потом окно выбора разворачивания контейнера ansible через docker (логика такая же как в manage_deploy.sh)
8. После подтверждение выбора и запуск деплоя (логи должны выводиться экране)
9. Когда все завершилось писать что все готово, если ошибка то писать что ошибка (+ логи всегда на экране)
10. Если все завершилось нормально спросить удалять ли репозиторий и если да то удалять

Для написания буду использовать Go wails

На шаге 3: Go-бэкенд делает обычный HTTP-запрос к GitHub по адресу: https://github.com/canntstand/ServeHub-2/archive/refs/heads/main.zip
Средствами Go (библиотека archive/zip) программа скачивает этот файл в память, распаковывает его в рабочую папку и удаляет сам .zip.
