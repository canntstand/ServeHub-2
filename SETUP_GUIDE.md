# Гайд по настройке инфраструктуры.
## ⚠️ Важно
1. **Поддерживаемые ОС:** Проект рассчитан на операционные системы, построенные на базе Linux (гарантирована работа на Ubuntu, Debian, Arch Linux).
2. **Настройка DNS:** В DNS-панели регистратора (например, на Webnames) необходимо направить A-запись основного домена и wildcard-запись (`*`) на внутренний IP-адрес VPN — **`10.8.0.1`**. Это критически важно: весь трафик к вашим сервисам должен идти внутри защищенного туннеля AmneziaWG. Если направить домен на публичный IP сервера VPS, Nginx заблокирует внешние запросы из-за правила `deny all`.

## Установка
### Linux
1. Установить:
    - **Docker** + **Docker Compose** 
    - **Git**
    - **SSH** 
2. Подготовить 2 Linux сервера с включенным ssh (Ubuntu/Debian/Arch)
3. Клонировать репозиторий: `git clone https://github.com/canntstand/ServeHub-2 && cd ServeHub-2`
4. Создать файл `ansible/vars/secrets.yml` на основе примера `ansible/vars/secrets.yml.example`.
5. Создать ssh ключ `ssh-keygen -t ed25519 -C "your_email@example.com"`
6. Запустить файл: `./manage_deploy.sh` (при первой установке на пустые сервера лучше выбрать 1 пункт)
7. Остальные шаги сообщит скрипт

### Windows
1. Установить:
    - **Docker** + **Docker Compose** 
    - **Git**
    - **SSH** 
    - **WSL + Дистрибутив**
    - **Терминал с поддержкой Bash**
2. Подготовить 2 Linux сервера с включенным ssh (Ubuntu/Debian/Arch)
3. Клонировать репозиторий: `git clone https://github.com/canntstand/ServeHub-2 && cd ServeHub-2`
4. Создать файл `ansible/vars/secrets.yml` на основе примера `ansible/vars/secrets.yml.example`.
5. Создать ssh ключ `ssh-keygen -t ed25519 -C "your_email@example.com"`
6. В WSL выполнить `mkdir -p ~/.ssh && cp /mnt/c/Users/ВАШЕ_ИМЯ_ПОЛЬЗОВАТЕЛЯ/.ssh/id_ed25519 ~/.ssh/ && chmod 600 ~/.ssh/id_ed25519` 
7. В WSL запустить файл (нужно перейти к корню проекта на Windows через /mnt/): `./manage_deploy.sh` (при первой установке на пустые сервера лучше выбрать 1 пункт)
8. Остальные шаги сообщит скрипт

## Пользование сервисами
1. **Вход на портал:** После успешного запуска перейдите по вашему домену на главную страницу (веб-портал), предварительно включив VPN-клиент AmneziaWG на вашем устройстве.
2. **Первичная настройка приложений:** Для сервисов Navidrome, Audiobookshelf, Vaultwarden и Portainer при первом открытии необходимо пройти быструю процедуру создания аккаунта администратора. Для доступа к системной панели администрирования Vaultwarden используется токен, заданный в переменной `secret_vaultwarden_password`.
3. **Автоматическое развертывание:** Сервисы Nextcloud, Grafana и другие настраиваются автоматически. Данные для первого входа в них соответствуют глобальным переменным `admin_user` и `admin_password` из вашего `secrets.yml`.
4. **Интеграция с Matrix:** Служебный аккаунт для Synapse (Matrix) создается автоматически. Скрипт деплоя направит вашей основной учетной записи инвайт в закрытую комнату. Примите его в клиенте Element, чтобы получать важные системные уведомления от Alertmanager.
5. **Рекомендованные приложения для повседневного использования:**
    - Облачное хранилище — *Nextcloud*
    - Менеджер паролей — *Bitwarden*
    - Мессенджер — *Element X*
    - Музыкальная коллекция — *Tempo*
    - Аудиокниги и подкасты — *Audiobookshelf*
    - VPN-клиент — *AmneziaWG*
6. **Прямое подключение по SSH внутри защищенного VPN-туннеля:**
    - К удаленному серверу (VPS): `ssh user@10.8.0.1`
    - К локальному серверу: `ssh user@10.8.0.2`
    - *(Обратите внимание: данные приватные IP-адреса будут доступны для подключения только тогда, когда на вашем текущем рабочем устройстве — ноутбуке или смартфоне — запущен и активен VPN-клиент AmneziaWG)*.
