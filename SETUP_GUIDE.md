⚠️ Важно: все команды выполняются из корня проекта, обращайте внимание на пометки (на VPS), (на локальном ПК) или (на клиенте).

## Развертывание Amnezia VPN (на внешнем VPS)
1. Скачать официальный клиент Amnezia VPN на ПК/смартфон.
2. В приложении выбрать «У меня есть данные для подключения» -> «Настроить свой сервер».
3. Ввести IP-адрес VPS, имя пользователя (`root` или вашего созданного пользователя) и пароль от VPS (или приватный SSH-ключ).
4. Выбрать уровень маскировки (рекомендуется протокол **AmneziaWG** или **XRay** для обхода жестких блокировок).
5. Нажать «Установить» и дождаться автоматического развертывания Docker-контейнеров внутри VPS.
6. После установки подключиться к созданному VPN и поделиться конфигурационным ключом/QR-кодом с нужными устройствами семьи.

## Подготовка
1. На клиенте (ПК/смартфон): Убедиться, что Amnezia VPN успешно развернута (по шагам выше) и включена на устройствах, с которых нужен доступ.
2. В DNS-панели вашего домена направить A-запись (и wildcard-запись *) на публичный IP-адрес вашего VPS. Выбрать любой свободный порт на VPS для проброса SSH наружу и указать его в переменную FRP_SSH_REMOTE_PORT в .env на локальной машине.
3. На локальном ПК: Установить argon2 (`sudo apt update && sudo apt install argon2 -y`) для генерации хеша пароля vaultwarden.
4. На VPS: Выполнить `git clone https://github.com/canntstand/Mini-PC-setup-FRP && cd Mini-PC-setup-FRP`. Создать там минимальный файл .env с одной переменной `FRP_TOKEN`.
5. На VPS: Запустить автоматическую настройку сетевой части одной командой: `chmod +x setup_remote.sh && ./setup_remote.sh`
6. На локальном ПК (домашнем сервере): Выполнить `git clone https://github.com/canntstand/Mini-PC-setup-FRP && cd Mini-PC-setup-FRP`.
7. На локальном ПК: Настроить все переменные в файле .env по примеру .env.example (в переменной `SYNAPSE_SERVER_NAME` обязательно указать твой домен, заполнить IP-адрес VPS в `FRP_SERVER_IP` и токен в `FRP_TOKEN`).
8. На локальном ПК (убедившись, что Amnezia VPN включена): Запустить сборку основного проекта одной командой: `chmod +x setup_local.sh && ./setup_local.sh`

## SSH (стоит настроить и на локальном сервере, и на удаленном vps)
1. ПРЕДУСТАНОВКА (Если вы зашли на чистый сервер под пользователем root):
   - Создайте нового пользователя: `sudo adduser server-user` (замените server-user на ваше имя)
   - Добавьте его в группу администраторов: `sudo usermod -aG sudo server-user`
   - Переключитесь на нового пользователя: `su - server-user`
   - ВАЖНО: Все дальнейшие шаги (генерация и копирование ключей) выполняются строго для этого созданного пользователя, а не для root!
2. `sudo systemctl enable --now ssh`
3. На клиенте (вашем ПК): `ssh-keygen -t ed25519 -C "servehub-access"`
4. На клиенте (вашем ПК): `ssh-copy-id server-user@server-ip` (ввести пароль сервера в последний раз; повторить для VPS и локального ПК).
5. Проверить вход без пароля с клиента. На сервере открыть конфигурацию: `sudo nano /etc/ssh/sshd_config`
6. Выставить и проверить параметры безопасности:
   - `PasswordAuthentication no`
   - `PermitRootLogin no`
   - `MaxAuthTries 3`
   - `PermitEmptyPasswords no`
7. Если вверху файла sshd_config есть строка `Include /etc/ssh/sshd_config.d/*.conf`, зайдите в файлы внутри этой папки и, если там присутствует параметр PasswordAuthentication, также переведите его в `no`.
8. `sudo systemctl restart ssh`
9. ВАЖНО: Не закрывая текущий терминал, открыть новое окно на ПК и проверить вход через `ssh user@server-ip`.
10. `chmod +x scripts/fail2ban_setup.sh && ./scripts/fail2ban_setup.sh`

## Добавление нового устройства к SSH (Например, телефона или рабочего ПК)
1. На новом устройстве: Сгенерировать ключ: `ssh-keygen -t ed25519 -C "new-device-access"`
2. Любым безопасным способом переслать строку из файла .pub нового устройства на ваш основной ПК или сервер.
3. На сервере: `nano ~/.ssh/authorized_keys`
4. Перейти в самый конец файла, создать новую строку, вставить туда публичный ключ нового устройства и сохранить файл.
5. Для удобства на ПК можно настроить `~/.ssh/config` (указать HostName — IP или домен вашего VPS, Port — это порт из переменной FRP_SSH_REMOTE_PORT и User), чтобы подключаться короткой командой `ssh home-server`.

## Пользование сервисами
После запуска перейдите по вашему домену на главную страницу (веб-портал), находясь под включенным Amnezia VPN. Оттуда можно открыть веб-интерфейс каждого сервиса.

Для Navidrome, Audiobookshelf и Vaultwarden при первом открытии в браузере необходимо пройти быструю процедуру создания аккаунта администратора.

Для Synapse (Matrix) аккаунт администратора создается автоматически в процессе деплоя.

Остальные сервисы являются служебными либо аккаунт для них создается автоматически с использованием переменных из .env.

Для повседневного использования рекомендуется установить официальные или сторонние приложения на ваш смартфон/ПК:

### Рекомендации (Сервис - Клиент)
1. Nextcloud — Официальный клиент Nextcloud
2. Vaultwarden — Bitwarden
3. Synapse (Matrix) — Element X (или Element)
4. Navidrome — Tempo / Amuse / Symfonium
5. Audiobookshelf — Официальное приложение Audiobookshelf
