# Гайд по настройке инфраструктуры.
## ⚠️ Важно
1. **Поддерживаемые ОС:** Проект рассчитан на работу на Ubuntu, Debian, Arch Linux.
2. **Настройка DNS:** В DNS-панели регистратора (например, на Webnames) необходимо направить A-запись основного домена и wildcard-запись (`*`) на внутренний IP-адрес VPN — **`10.8.0.1`**. Это критически важно: весь трафик к вашим сервисам должен идти внутри защищенного туннеля AmneziaWG. Если направить домен на публичный IP сервера VPS, Nginx заблокирует внешние запросы из-за правила `deny all`.
3. **VPN для Vagrant:** HashiCorp в данный момент не работает в России, поэтому для запуска тестирования понадобится VPN. (после установки машин VPN следует выключить, либо можно включить split-tunneling на игнорирование 192.168.0.0/24, так как иначе к тестовым виртуалкам не получится подключиться по ssh)
4. **Сохранность бэкапов:** После настройки локального сервера стоит сохранить ключ borg (если настраивалась система бэкапов): `borg key export /mnt/backup_storage/server_backup.borg /home/USER/borg_key_backup.txt`. (лучше перенести в другое место, не стоит хранить на том же сервере)
5. **Настройка wg-easy:** Если нет подключения к VPN, значит вероятно нужно зайти в веб интерфейс и настроить S3-S4 и I1-I5 параметры. (подробнее можно прочитать в [AmneziaWG documentation](https://docs.amnezia.org/documentation/amnezia-wg))

## Установка
### Linux
1. Установить:
    - **Docker** + **Docker Compose** 
    - **Git**
    - **SSH**
2. Подготовить 2 Linux сервера с включенным ssh (Ubuntu/Debian/Arch)
3. Клонировать репозиторий: `git clone https://github.com/canntstand/ServeHub-2 && cd ServeHub-2`
4. Создать файл `ansible/vars/secrets.yml` на основе примера `ansible/vars/secrets.yml.example`. (для генерации паролей можно использовать `./scripts/generate_secrets.py`)
5. Сгенерировать SSH-ключ: `ssh-keygen -t ed25519 -C "your_email@example.com"`
6. Запустить скрипт развертывания: `chmod +x manage_deploy.sh && ./manage_deploy.sh` (при первой установке на пустые сервера лучше выбрать 1 пункт)
7. Остальные шаги сообщит скрипт

### Windows
1. Установить:
    - **Docker** + **Docker Compose** 
    - **Git**
    - **SSH** 
    - **WSL + Дистрибутив**
    - **Терминал с поддержкой Bash**
2. Подготовить 2 Linux сервера с включенным ssh (Ubuntu/Debian/Arch)
3. Зайти в терминал дистрибутива в WSL (все остальные пункты будут выполняться строго в нем)
4. Клонировать репозиторий (где угодно в /mnt/c/): `git clone https://github.com/canntstand/ServeHub-2 && cd ServeHub-2`
5. Создать файл `ansible/vars/secrets.yml` на основе примера `ansible/vars/secrets.yml.example`. (для генерации паролей можно использовать `./scripts/generate_secrets.py`)
6. Сгенерировать SSH-ключ и выдать нужные права: `ssh-keygen -t ed25519 -C "your_email@example.com" && chmod 600 ~/.ssh/id_ed25519`
7. Запустить скрипт развертывания: `chmod +x manage_deploy.sh && ./manage_deploy.sh` (при первой установке на пустые сервера лучше выбрать 1 пункт)
8. Остальные шаги сообщит скрипт.

### Настройка бэкапов (доп. к стандартной установке)
1. Убедиться, что диск для бэкапов подключен к серверу.
2. Указать UUID диска в secrets.yml. (можно посмотреть с помощью `sudo blkid`)
3. После разворачивания локальной архитектуры зайти в Borg UI (username всегда admin) и выполнить следующие шаги:
    1. Выбрать импорт репозитория (указать имя /mnt/backup_storage/server_backup.borg)
    2. Выбрать Observability Only и Read-only storage access (бэкапами занимается borgmatic, Borg UI только для просмотра)
    3. Указать passphrase которая была указана в secrets.yml.
    4. Выбрать тип сжатия lz4.
    5. Импортировать репозиторий.
4. **Напоминалка:** бэкапы через Borg UI нельзя настраивать ни в коем случае!

### Настройки автономности локального сервера
1. Для того, чтобы сервер включался обратно после отключения электричества в BIOS нужно найти настройки: (пишу как назывались у меня, имена могут отличаться)
    1. **State After G3:** нужно выбрать `S0 State`
    2. **Wake system from S5:** `Disabled`

### Тестирование с Vagrant
1. Установить (дополнительно):
    - **Vagrant**
    - **VirtualBox**
2. На основной системе в корне проекта выполнить `vagrant up` для создания серверов для тестирования.
    - На вопрос **Which interface should the network bridge to?** нужно выбрать тот интерфейс, через который хост выходит в интернет.
3. В остальном все по стандартному гайду.
4. Чтобы остановить виртуальные машины можно использовать `vagrant halt`. (остальные команды можно посмотреть с помощью `vagrant --help`)

## Пользование сервисами
1. **Вход на портал:** После успешного запуска перейдите по вашему домену на главную страницу (веб-портал), предварительно включив VPN-клиент AmneziaWG на вашем устройстве.
2. **Первичная настройка приложений:** Для сервисов Navidrome, Audiobookshelf, Vaultwarden и Portainer при первом открытии необходимо пройти быструю процедуру создания аккаунта администратора. Для доступа к системной панели администрирования Vaultwarden используется токен, заданный в переменной `secret_vaultwarden_password`.
3. **Автоматическое развертывание:** Сервисы Nextcloud, Grafana и другие настраиваются автоматически. Данные для первого входа в них соответствуют глобальным переменным `admin_user` и `admin_password` из вашего `secrets.yml`.
4. **Интеграция с Matrix:** Служебный аккаунт для Synapse (Matrix) создается автоматически. Скрипт деплоя направит вашей основной учетной записи инвайт в закрытую комнату. Примите его в клиенте Element, чтобы получать важные системные уведомления от Alertmanager.
5. **Рекомендованные приложения для повседневного использования:**
    - Облачное хранилище — *Nextcloud*
    - Менеджер паролей — *Bitwarden*
    - Мессенджер — *Element X*
    - Музыкальная коллекция — *Tempo*
    - Аудиокниги и подкасты — *Audiobookshelf*
    - VPN-клиент — *AmneziaWG*
6. **Прямое подключение по SSH внутри защищенного VPN-туннеля:**
    - К удаленному серверу (VPS): `ssh user@10.8.0.1`
    - К локальному серверу: `ssh user@10.8.0.2`
    - *(Обратите внимание: данные приватные IP-адреса будут доступны для подключения только тогда, когда на вашем текущем рабочем устройстве — ноутбуке или смартфоне — запущен и активен VPN-клиент AmneziaWG)*.
7. **Использование папок:** (при переносе проекта главное скопировать эти папки)
    - В папке `~/ServeHub-2/apps-data` хранятся данные приложений - при переносе проекта ее важно сохранить.
    - В папке `~/PersonalData` хранятся все пользовательские данные: документы, фото, видео музыка и тд.
